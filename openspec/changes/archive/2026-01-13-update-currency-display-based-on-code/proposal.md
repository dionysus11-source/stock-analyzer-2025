# Change: 주식 코드에 따른 통화 표시 변경

## Why
현재 모든 주식의 평가손익이 '원'으로 고정되어 표시됩니다. CSV 데이터에 미국 주식(영문 코드)과 한국 주식(숫자 코드)이 섞여 있을 경우, 통화 표시가 올바르지 않아 사용자에게 혼동을 줍니다.

## What Changes
- '코드' 열의 값을 확인하여 통화를 동적으로 변경합니다.
  - '코드'가 문자로만 이루어져 있으면(미국 주식으로 간주), 평가손익을 달러($)로 표시합니다.
  - '코드'가 숫자로만 이루어져 있으면(한국 주식으로 간주), 평가손익을 원(KRW)으로 표시합니다.
- `DataAnalyzer`가 분석 결과에 통화 정보를 포함하여 반환하도록 수정합니다.
- `MainUI`가 이 통화 정보를 사용하여 평가손익 및 총 평가손익을 올바른 기호($, 원)로 표시하도록 수정합니다.

## Impact
- **Affected Specs:**
  - `data-analyzer` (new)
  - `ui-components` (modified)
- **Affected Code:**
  - `data_analyzer.py`
  - `ui_components.py`
