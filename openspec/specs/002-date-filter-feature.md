# OpenSpec: 분석 데이터 날짜 필터링 기능 추가

## 1. 제안 배경 (Background)

- 현재는 CSV 파일 전체를 분석 대상으로 하여, 특정 기간의 데이터만 보는 것이 불가능합니다.
- 사용자가 원하는 기간의 '매도' 내역만 필터링하여 분석의 유연성과 편의성을 높일 필요가 있습니다.

## 2. 제안 내용 (Proposal)

- 분석 시작일과 종료일을 입력받아, 해당 기간 내의 '매도' 데이터에 대해서만 분석을 수행하는 기능을 추가합니다.
- 날짜를 입력하지 않을 경우, 기존과 동일하게 전체 데이터를 분석합니다.

## 3. 기능 명세 (Specification)

### 3.1. 사용자 인터페이스 (UI)

- `ui_components.py`의 '파일 선택 및 병합' 영역에 다음을 추가합니다:
    - '시작일' `ttk.Label` 및 `ttk.Entry` 위젯
    - '종료일' `ttk.Label` 및 `ttk.Entry` 위젯
- 각 입력 상자 옆에는 `(YYYY-MM-DD)` 형식으로 입력을 유도하는 안내 문구를 표시합니다.

### 3.2. 데이터 필터링 프로세스

1.  **사용자 입력**: 사용자가 '시작일' 및/또는 '종료일'을 입력하고 '분석 시작' 버튼을 클릭합니다.
2.  **입력 확인**:
    - 날짜가 입력되지 않은 경우: 기존 분석 로직을 그대로 수행합니다.
    - 날짜가 입력된 경우: 아래의 필터링 로직을 수행합니다.
3.  **필터링 로직 (`data_analyzer.py` 수정)**:
    - CSV 파일을 `pandas` DataFrame으로 읽어들입니다.
    - **'매매구분' 열의 값이 '매도'인 행만 선택**하여 새로운 DataFrame을 생성합니다.
    - **'날짜' 열의 데이터 (`YYYY.MM.DD` 형식)를 datetime 객체로 변환**합니다.
    - 사용자가 입력한 시작일/종료일 기준으로 '날짜' 열을 필터링합니다. (예: `start_date <= df['날짜'] <= end_date`)
    - 최종 필터링된 DataFrame을 대상으로 평가손익 분석을 수행합니다.

## 4. 기대 효과 (Expected Outcome)

- 사용자가 원하는 특정 기간의 손익 현황을 쉽게 파악할 수 있습니다.
- 분석의 범위 조절이 가능해져 사용성이 증대됩니다.
